### Commit Convention

| 타입 | 의미 |
| :---: | :--- |
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| docs | 문서(README 등) 수정 |
| style | 코드 포맷 변경 (세미콜론, 들여쓰기 등), 코드 의미 변경 없음 |
| refactor | 코드 리팩토링 (동작 변화 없음) |
| test | 테스트 코드 추가 또는 수정 |
| chore | 빌드 업무, 패키지 설정, 환경 설정 변경 |
| build | 빌드 시스템 또는 외부 의존성 변경 (예: 패키지 추가/업데이트) |
